# YoloLabel

Десктоп-приложение для разметки датасетов object detection в формате YOLO.  
Написано на Python + PyQt6, работает на Windows.

---

## Для чего

YoloLabel предназначен для ручной разметки изображений и подготовки датасетов для обучения моделей YOLO (YOLOv5/v8/v11 и совместимых). Приложение читает стандартный `data.yaml`-конфиг датасета и работает напрямую с файлами аннотаций в текстовом формате YOLO (`.txt`).

**Основные задачи:**
- Рисование и редактирование bounding box и полигональных сегментаций
- Просмотр и корректировка уже существующих аннотаций
- Распределение изображений по сплитам (train / val / test / review)
- Добавление изображений перетаскиванием (drag-and-drop)
- Предобработка изображений перед обучением (кадрирование, поворот, мозаика)
- Предпросмотр изображения в масштабе нейронной сети

## Установка

**Требования:** Python 3.10+, Windows, venv.

```bash
# Клонировать репозиторий
git clone <repo-url>
cd yololabel

# Создать виртуальное окружение и установить зависимости
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
```

**Зависимости** (`requirements.txt`):
- `PyQt6 >= 6.4` — GUI
- `PyYAML >= 6.0` — чтение конфигов датасета
- `python-dotenv >= 0.19` — переменные окружения (опционально, для разработки)

---

## Запуск

```bash
# Активировать venv (если не активирован)
.venv\Scripts\activate

# Запустить приложение
python main.py
```

После запуска открыть датасет через кнопку с иконкой папки (или выбрать из истории последних конфигов) — выбрать файл `data.yaml`.

## Архитектура

```
yololabel/
├── main.py                   # Точка входа, создание QApplication
├── requirements.txt
├── core/                     # Бизнес-логика, без зависимости от PyQt6
│   ├── models.py             # Типы данных: Annotation, AnnType
│   ├── config_io.py          # Чтение data.yaml, резолвинг путей сплитов
│   ├── annotation_io.py      # Чтение/запись .txt YOLO-аннотаций
│   └── image_ops.py          # Пересчёт аннотаций при crop/rotate
└── gui/
    ├── theme.py              # Тема оформления (Fusion + dark palette)
    ├── style.py              # Design-токены и QSS-стилевой лист
    ├── main_window.py        # Главное окно, вся логика UI
    ├── app_icon.py           # Генерация иконки приложения
    └── editor/
        ├── viewer.py         # QGraphicsView — отображение и взаимодействие с изображением
        ├── items.py          # QGraphicsItem для bbox и сегментаций
        └── image_list_widget.py  # Список изображений с фоновой загрузкой превью
```

**Принцип разделения:** модули в `core/` не импортируют PyQt6 — их можно использовать в скриптах и тестах без GUI. Модули в `gui/` содержат весь код интерфейса и могут зависеть от `core/`.

## Функционал

### Разметка

| Действие | Способ |
|---|---|
| Нарисовать bbox | Кнопка «✏ Рамка» или `Пробел`, затем drag |
| Выделить несколько | `Shift + клик` или rubber-band drag |
| Переместить bbox | Drag по центру |
| Изменить размер | Drag за маркеры по углам и сторонам |
| Сдвинуть на 1px | Стрелки; `Shift + стрелки` — на 10px |
| Сменить класс | Выпадающий список Class или двойной клик в списке аннотаций |
| Удалить аннотацию | `Del` или кнопка «Удалить рамку» |
| Конвертировать сег → bbox | Двойной клик по сегментации или кнопка «→ BBox» |
| Выделить все | `Ctrl+A` |

### Навигация и сохранение

| Действие | Способ |
|---|---|
| Следующее/предыдущее изображение | `←` / `→` или кнопки `‹` / `›` |
| Сохранить аннотации | `Ctrl+S` или иконка 💾 в шапке панели |
| Автосохранение | Чекбокс «Автосохранение» в тулбаре |
| Undo / Redo (60 уровней) | `Ctrl+Z` / `Ctrl+Y` |
| Дублировать изображение | `C` или кнопка ⧉ в шапке панели |
| Добавить изображение | Перетащить файл в область просмотра или список изображений |
| Fit изображения в экран | Средняя кнопка мыши или `0` |
| Панорамирование | Правая кнопка мыши + drag |
| Зум | `колесо` — масштаб; `Ctrl + колесо` — размер меток |

### Предобработка изображений

| Действие | Способ |
|---|---|
| Кадрирование | Кнопка «✂» или `X`, затем `Enter` для подтверждения |
| Поворот по часовой | `R` |
| Поворот против часовой | `Shift+R` |
| Мозаика N×N | `M` или кнопка «⊞ Mosaic» — составляет сетку из текущего изображения; повторное нажатие увеличивает сетку (до 8×8), `Esc` — отмена |
| Яркость/Контраст | Слайдеры B/C в правой панели (display-only, файл не изменяется) |
| Предпросмотр в масштабе нейросети | Чекбокс «Размер нейросети» + спиннер с размером входа (px) |

Crop, rotate и мозаика сохраняются на диск при нажатии `Ctrl+S` и пересчитывают координаты всех аннотаций.

### Управление датасетом

**Сплиты.** Кнопки `● Train 1` / `● Val 2` / `● Test 3` / `⚡ Отложить 4` перемещают текущее изображение (вместе с файлом аннотации) в соответствующую папку сплита. Если папка сплита не существует — приложение предлагает создать её. Папка `review/` создаётся автоматически при первом использовании и подхватывается при следующих открытиях датасета.

Чекбокс **«Перенос»** (справа от «Отложить») включает/выключает фактическое перемещение файла. Когда снят — кнопки 1/2/3/4 только листают изображения без перемещения файлов. Если изображение уже находится в целевом сплите — автоматически выполняется переход к следующему.

Переключение между сплитами — через выпадающий значок текущего сплита справа на панели (показывает количество изображений в каждом сплите).

**Статусы изображений.** Каждое изображение помечается цветом в списке:
- серый — непросмотренное
- белый — просмотренное
- зелёный — сохранённое

Статус меняется через контекстное меню (ПКМ по элементу списка) или двойным `Esc` (сброс в «непросмотрено»).

**Удаление файла.** Иконка 🗑 в шапке панели «ИЗОБРАЖЕНИЯ» или клавиша `D` удаляет текущий файл изображения и его аннотацию с диска.

**Добавление изображений перетаскиванием.** Перетащите один или несколько файлов изображений в область просмотра или в список изображений — файлы будут скопированы в папку текущего сплита и сразу открыты. Поддерживаемые форматы: `.jpg`, `.jpeg`, `.png`, `.bmp`, `.tiff`, `.webp`. При конфликте имён появляется диалог с атрибутами обоих файлов (размер, дата, разрешение): можно заменить существующий файл, сохранить под новым именем или пропустить.

### Прочее

- **История конфигов** — кнопка ▾ рядом с иконкой папки хранит последние 10 открытых датасетов; удаление записи — клавиша `Del` в меню
- **Настройки** — кнопка ⚙ меняет назначение кнопок мыши (pan / fit), настройку подтверждения удаления и шаг мозаики
- **Размер меток** — `Ctrl + колесо мыши` над холстом

## Формат данных

Приложение работает со стандартным форматом датасета YOLO:

```
dataset/
├── data.yaml          # Конфиг: nc, names, train/val/test пути
├── train/
│   ├── images/        # .jpg / .png / .bmp / .webp / .tiff
│   └── labels/        # .txt файлы в формате YOLO
└── val/
    ├── images/
    └── labels/
```

Файл `data.yaml` должен содержать поля `nc` (число классов), `names` (список или словарь имён) и пути `train`, `val`, `test` (опционально). Пути могут указывать на папки с любым именем — например, `val: ../valid/images` корректно обрабатывается. Все сплиты из `data.yaml` подхватываются автоматически.
